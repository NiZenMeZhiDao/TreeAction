# MF Action Planner

一个基于 ROS2 的梅林区路径、操作规划器，支持Web界面交互。
仓库地址：[https://github.com/BetterOIer/mf_action_planner.git](https://github.com/BetterOIer/mf_action_planner.git)

## 参数详情

### 关于地图：

始终令武馆区右侧第一个台阶前方的格子为 `(0, 0)`，以此为基准进行坐标定义。

```
蓝区：
+-------------+-------------+-------------+-----------------+
| (5, 2, 000) | (5, 1, 000) | (5, 0, 000) |    ←出口区坐标
+=============+=============+=============+=================+
║ (4, 2, 200) | (4, 1, 400) | (4, 0, 200) ║
+-------------+-------------+-------------+
║ (3, 2, 400) | (3, 1, 600) | (3, 0, 400) ║
+-------------+-------------+-------------+    ←梅林区坐标
║ (2, 2, 200) | (2, 1, 400) | (2, 0, 600) ║
+-------------+-------------+-------------+
║ (1, 2, 400) | (1, 1, 200) | (1, 0, 400) ║
+=============+=============+=============+=================+
| (0, 2, 000) | (0, 1, 000) | (0, 0, 000) |    ←入口区坐标
+-------------+-------------+-------------+-----------------+

红区：
+-------------+-------------+-------------+-----------------+
| (5, 2, 000) | (5, 1, 000) | (5, 0, 000) |    ←出口区坐标
+=============+=============+=============+=================+
║ (4, 2, 200) | (4, 1, 400) | (4, 0, 200) ║
+-------------+-------------+-------------+
║ (3, 2, 400) | (3, 1, 600) | (3, 0, 400) ║
+-------------+-------------+-------------+    ←梅林区坐标
║ (2, 2, 600) | (2, 1, 400) | (2, 0, 200) ║
+-------------+-------------+-------------+
║ (1, 2, 400) | (1, 1, 200) | (1, 0, 400) ║
+=============+=============+=============+=================+
| (0, 2, 000) | (0, 1, 000) | (0, 0, 000) |    ←入口区坐标
+-------------+-------------+-------------+-----------------+

↑为弧度0方向，逆时针为正，范围[-π, π]
```

该规划器会使用ROS2 `topic`发送上述**网格坐标**（row, col），接受者应使用高度地图将其转化为**实际地图坐标**进行处理。


### 关于topic

名称：`/mf_action_seq`
类型：`std_msgs/Float32MultiArray`
内容：规划梅林区的行动序列
`[arg0, arg1, arg2, arg3, arg4, arg5, arg6, arg7]`

`arg0`状态参数：`0` 为 `move`, `1` 为 `fetch`

`move`状态下：
- `arg1`：目标格子行 row (0~5)
- `arg2`：目标格子列 col (0~2)
- `arg3`：目标高度 (mm: 0/200/400/600)
- `arg4`：目标位置朝向（弧度）

`fetch`状态下：
- `arg1`：目标格子行 row (0~5)
- `arg2`：目标格子列 col (0~2)
- `arg3`：目标位置高度差,目标减去当下
- `arg4`：目标位置朝向（弧度）

`arg5`-`arg7`：保留参数，暂未使用
